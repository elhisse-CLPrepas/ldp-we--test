# Audit du cahier des charges

Source contrôlée : `guide-source-ldp.docx`.

## Résultat

Le projet respecte les exigences principales du cahier des charges pour une première version publiable.

## Points conformes

- Projet : LAB-NUM-CTM - Laboratoire Numérique CTM.
- Page statique HTML5 avec CSS séparé et JavaScript séparé.
- Chemins relatifs uniquement dans la page.
- Structure demandée : `index.html`, `README.md`, `.gitignore`, `assets/css/style.css`, `assets/js/app.js`, `assets/img/`, `data/faq.json`, `data/features.json`, `docs/guide-utilisation.md`.
- Sections obligatoires présentes : Hero, Problème, Vision, Outils, Méthode, Production, FAQ, Appel à l’action, Footer CTM.
- Outils couverts : VS Code, Codex, Git, GitHub et navigateur.
- Méthode couverte : idée, production, test, commit, publication.
- Logo, affiches, image hero et image Open Graph intégrés.
- SEO de base et balises Open Graph ajoutés.
- Style sobre, professionnel, pédagogique et responsive.
- Données FAQ et outils externalisées en JSON avec contenu statique de secours.

## Points à personnaliser avant publication

- Remplacer le `mailto:` sans destinataire par l’adresse officielle ou un lien d’inscription.
- Ajouter l’URL publique finale dans `og:url` lorsque la page sera publiée.
- Créer les commits Git aux étapes importantes, puis publier sur GitHub.
